# Color Palettes for Tech Blogs

## Light Mode
Professional, clean aesthetic with strong contrast for readability.

```
Primary:      #1a1a1a (near-black text)
Secondary:    #666666 (muted text)
Accent:       #0066ff (electric blue)
Background:   #ffffff (white)
Surface:      #f5f5f5 (light gray)
Border:       #e0e0e0 (subtle gray)
Code BG:      #f8f8f8 (near-white with tint)
Link Hover:   #0052cc (darker blue)
Success:      #28a745 (github green)
Warning:      #ffc107 (amber)
Error:        #dc3545 (red)
```

## Dark Mode
Inspired by modern tech platforms (GitHub Dark, VS Code).

```
Primary:      #ffffff (white text)
Secondary:    #b0b0b0 (light gray text)
Accent:       #4da6ff (softer blue)
Background:   #0d1117 (deep navy-black)
Surface:      #161b22 (elevated surface)
Border:       #30363d (subtle dark border)
Code BG:      #0d1117 (matches background)
Link Hover:   #79c0ff (bright blue)
Success:      #3fb950 (github green)
Warning:      #d29922 (darker amber)
Error:        #f85149 (soft red)
```

## Accessibility Considerations

- Light mode text (#1a1a1a on #ffffff) = WCAG AAA compliant
- Dark mode text (#ffffff on #0d1117) = WCAG AAA compliant
- Accent colors meet AA standard for UI components
- Avoid pure black (#000000) and pure white (#ffffff) combinations for reduced eye strain

## Usage Guidelines

**Primary color:** Body text, main content
**Secondary color:** Metadata, timestamps, less important info
**Accent color:** Links, buttons, highlights, focus states
**Background:** Page foundation
**Surface:** Cards, code blocks, elevated areas
**Border:** Dividers, form inputs
**Semantic colors:** Status indicators (success/warning/error)
